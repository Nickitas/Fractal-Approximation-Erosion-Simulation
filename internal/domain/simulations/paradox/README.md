# Package `paradox`

**Демонстрация парадокса береговой линии через масштабную зависимость и фрактальное обогащение геометрии.**

Модуль наглядно показывает, что длина береговой линии не является абсолютной величиной — она зависит от масштаба измерения. При каждом уменьшении шага появляются новые геометрические детали, и длина неограниченно растёт, хотя кривая остаётся в ограниченной области плоскости.

## Научная постановка

### Парадокс береговой линии

Парадокс береговой линии — фундаментальное наблюдение: **длина береговой линии стремится к бесконечности при неограниченном уменьшении масштаба измерения**, несмотря на то что береговая линия ограничена конечной областью.

**Формально:**

```
L(ε) → ∞  при  ε → 0

где:
  ε — масштаб измерения (длина «линейки»)
  L(ε) — измеренная длина береговой линии
```

Это противоречит интуитивному представлению о длине как о фиксированной характеристике объекта.

**Практический пример:**

| Масштаб «линейки» | Измеренная длина |
|-------------------|------------------|
| 200 км | ~4000 км |
| 50 км | ~5200 км |
| 10 км | ~6800 км |
| 1 км | ~10000+ км |

### Ричардсон и Мандельброт

Парадокс был впервые описан **Льюисом Фрайем Ричардсоном** (1961) и популяризирован **Бенуа Мандельбротом** в знаменитой статье:

> Mandelbrot, B. B. (1967). *How Long Is the Coast of Britain? Statistical Self-Similarity and Fractional Dimension.* Science, 156(3775), 636–638.

Мандельброт показал, что береговые линии обладают **статистическим самоподобием** и могут быть охарактеризованы **фрактальной размерностью** `D`:

```
L(ε) ∝ ε^(1-D)

где D ∈ (1, 2) — фрактальная размерность
```

Для Великобритании `D ≈ 1.25`, что близко к размерности кривой Коха (`≈ 1.26`).

## Математическая модель

### Масштабная зависимость длины

Демонстрация использует два механизма увеличения длины:

**1. Уменьшение шага измерения** — при более мелком масштабе обнаруживаются новые детали рельефа.

**2. Геометрическое обогащение** — каждая итерация Коха заменяет один сегмент на четыре, добавляя новые детали:

```
Итерация 0:  A──────────────────B       (1 сегмент, длина L₀)
Итерация 1:  A────╱╲────B               (4 сегмента, длина L₀ × 4/3)
Итерация 2:  A──╱╲──╱╲──╱╲──B          (16 сегментов, длина L₀ × 16/9)
```

### Кривая Коха как механизм обогащения

На каждой итерации `n`:

```
Число сегментов:  S(n) = 4ⁿ × S₀
Длина сегмента:   l(n) = l₀ / 3ⁿ
Общая длина:      L(n) = L₀ × (4/3)ⁿ
Средний шаг:      ε(n) = L(n) / S(n) = l₀ / 3ⁿ × 4ⁿ / 4ⁿ = l₀ / 3ⁿ × (3/4)ⁿ × (4/3)ⁿ

Упрощённо:        ε(n) ≈ L₀ / S(n)
```

**Рост длины:**

| Итерация `n` | Множитель длины | Рост |
|--------------|-----------------|------|
| 0 | `(4/3)⁰ = 1.000` | Базовая |
| 1 | `(4/3)¹ = 1.333` | +33% |
| 2 | `(4/3)² = 1.778` | +78% |
| 3 | `(4/3)³ = 2.370` | +137% |
| 4 | `(4/3)⁴ = 3.160` | +216% |
| 5 | `(4/3)⁵ = 4.214` | +321% |
| 10 | `(4/3)¹⁰ ≈ 17.76` | +1676% |

### Влияние эрозии

При `erosionStrength > 0` к каждой итерации применяется гауссовский сдвиг точек:

```
curveₙ = ErodeWithSeed(KochCurve(base, n), strength, seed + n)
```

**Эффект:**
- Случайные смещения добавляют микронеровности
- Длина увеличивается дополнительно к теоретическому росту Коха
- Форма становится более «природной»
- При фиксированном seed результат воспроизводим

**Почему `seed + level`?** Разный seed для каждой итерации обеспечивает независимые случайные сдвиги на каждом уровне, сохраняя общую детерминированность.

## Алгоритм демонстрации

```
Demonstrate(base, maxIterations, erosionStrength, seed)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Вывод заголовка и описания

2. Для level = 0..maxIterations:
   a. curve = KochCurve(base, level)
   
   b. Если erosionStrength > 0:
      curve = ErodeWithSeed(curve, erosionStrength, seed + level)
   
   c. length = PolylineLength(curve)
   
   d. segments = max(len(curve) - 1, 0)
   
   e. avgStep = length / segments  (если segments > 0)
   
   f. Если level > 0:
      growth = length - prevLength
      ratio = length / prevLength
      Вывести: length, growth, ratio
   Иначе:
      Вывести: length (без прироста)
   
   g. prevLength = length

3. Вывод заключения: длина растёт за счёт новых геометрических деталей
```

**Консольная таблица:**

```
Уровень   Точек        Сегментов        Средний шаг, км     Длина, км
──────────────────────────────────────────────────────────────────────────
0        242          241              18.54              4478 | —
1        964          963              6.18               5971 | +1493 км (1.333x)
2        3852         3851             2.06               7961 | +1991 км (1.333x)
...
```

## Публичный API

| Функция | Описание | Возвращает |
|---------|----------|------------|
| `Demonstrate(base, maxIter, erosionStrength, seed)` | Консольная демонстрация парадокса | `void` (вывод в stdout) |

**Параметры:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `base` | `[]geometry.LatLon` | Базовая полилиния береговой линии |
| `maxIterations` | `int` | Максимальное число итераций Коха |
| `erosionStrength` | `float64` | σ гауссовского сдвига в метрах (0 = без эрозии) |
| `seed` | `int64` | Seed для воспроизводимости (0 = использовать текущее время) |

## Примеры использования

### Базовая демонстрация без эрозии

```go
package main

import (
    "coastal-geometry/internal/domain/coastline"
    "coastal-geometry/internal/domain/simulations/paradox"
)

func main() {
    result, err := coastline.Load(coastline.LoadOptions{
        LocalPath: "data/black-sea.json",
    })
    if err != nil {
        panic(err)
    }
    
    // 5 итераций Коха, без эрозии
    paradox.Demonstrate(result.Points, 5, 0.0, 0)
}
```

### Демонстрация с эрозией

```go
func main() {
    result, err := coastline.Load(coastline.LoadOptions{
        LocalPath: "data/black-sea.json",
    })
    if err != nil {
        panic(err)
    }
    
    // 5 итераций + эрозия σ=50м, воспроизводимый seed
    paradox.Demonstrate(result.Points, 5, 50.0, 42)
}
```

### Программный доступ к данным

Для программного получения метрик (без консольного вывода) используйте `koch.KochCurve` напрямую:

```go
func main() {
    points, _, _ := coastline.LoadFromJSON("data/black-sea.json")
    
    fmt.Println("Итерация | Точек | Длина, км | Прирост")
    fmt.Println("─────────┼───────┼───────────┼────────")
    
    prevLength := 0.0
    for iter := 0; iter <= 5; iter++ {
        curve := koch.KochCurve(points, iter)
        length := geometry.PolylineLength(curve)
        
        growth := ""
        if iter > 0 {
            growth = fmt.Sprintf("+%.0f (%.3fx)", 
                length-prevLength, length/prevLength)
        }
        
        fmt.Printf("%-9d| %-6d| %-10.0f| %s\n",
            iter, len(curve), length, growth)
        
        prevLength = length
    }
}
```

```
--------------------------------------------------------------------------------
0          242          241              18.54              4478 | —
1          964          963              6.18               5971 | +1493 км (1.333x)
2          3852         3851             2.06               7961 | +1991 км (1.333x)
3          15404        15403            0.69               10615 | +2654 км (1.333x)

5          246444       246443           0.08               18871 | +4718 км (1.333x)

```

**Наблюдения:**

1. **Длина растёт экспоненциально** — множитель `≈ 1.333 = 4/3` на каждой итерации
2. **Средний шаг уменьшается** — в 3 раза на каждой итерации (18.54 → 6.18 → 2.06 → ...)
3. **Число точек растёт в 4 раза** — 242 → 964 → 3852 → ... (за исключением первой и последней)
4. **При n → ∞ длина → ∞** — но кривая остаётся в ограниченной области
